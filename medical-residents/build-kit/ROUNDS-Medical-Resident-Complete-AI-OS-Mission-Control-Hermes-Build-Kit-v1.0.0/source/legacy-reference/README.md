# ROUNDS — Medical Resident Complete AI OS

This package is a standalone, resident-only Hermes AI OS for postgraduate physicians in training. It uses the `medical_resident` lane, `/medical-residents` route, `medres_rounds.*` namespace, and one resident home named **My ROUNDS**. It must not be installed inside, under, or as a shared extension of a nursing population.

## Install this one file

Give Hermes the complete file:

`Medical-Resident-Complete-AI-OS-with-ROUNDS-SuperPowers-Hermes-Program.md`

Use the whole file. Do not split, reorder, or separately install the development-source components.

## Allow time

This is a substantial installation and may require several Hermes turns. Nothing continues in the background. Keep the S0, S1, and S2 receipts and wait for the final **ROUNDS Activation Report** before treating setup as complete. A dashboard preview or interim receipt is progress, not completion.

If Hermes cannot prove that the target is a separate Medical Resident workspace, installation must stop before mutation and request a separate workspace.

## Safe starting state

- Private Resident OS; no PHI or patient-specific live-care content
- Session-only memory; connectors and external actions Off
- All 24 SuperPowers **Available Inactive**
- All 10 suggested software agents **Disabled**
- One My ROUNDS home with the Core Four and an empty optional fifth launcher
- ATTEND authority checks, CIRCLE care-orchestration safeguards, and ORBIT agent governance
- Pause, kill, rollback, purge, removal, and full-uninstall controls

Installing ROUNDS does not grant clinical authority, change supervision, certify an institutional workspace, or replace local GME, specialty, board, privacy, safety, quality, research, or medical-staff requirements.

## First use

After the final Activation Report, begin with resident-owned nonsensitive information, public sources, and clearly synthetic scenarios. Start with zero or one low-risk power. Preview it, verify the human owner, source, supervision context, data boundary, expiry, fallback, and removal path, then choose whether to activate it. Activating none is acceptable.

For patient-specific, institutional, quality, research, education, or other sensitive work, use only a separately approved institutional environment with exact human authorization, minimum-necessary data, official systems of record, logging, review, retention, deletion, and incident routes.
