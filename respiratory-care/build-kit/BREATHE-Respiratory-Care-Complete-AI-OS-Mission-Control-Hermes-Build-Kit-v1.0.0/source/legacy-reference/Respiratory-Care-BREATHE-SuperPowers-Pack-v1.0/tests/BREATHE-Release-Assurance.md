# BREATHE Release Assurance
**Release ledger:** 72 foundation-domain checks (A–I) + 72 SuperPower-domain checks (J–R) + 16 integration checks = **160 total**.
## Evidence-bearing ledger contract

Every requirement produces a record with `check_id`, `requirement`, `method`, `expected_evidence`, `actual_evidence_refs`, `environment_and_version`, `tested_by`, `tested_at`, `result` (`Pass`, `Fail`, `Blocked` or `Not Run`), `criticality`, `blocker`, `remediation_owner`, `retest_at` and `receipt_id`. A checked box, assertion, missing log or self-report is not evidence. `Pass` is unavailable until actual evidence matches the expected evidence. Any critical `Fail`, `Blocked` or `Not Run` blocks release.

Synthetic prohibited-data markers must be unmistakably fictional and contain no real secret or health information. Integration tests repeat the highest-risk negative probes after all components are installed.
