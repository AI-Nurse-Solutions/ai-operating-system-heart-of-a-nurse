# THRIVE — Wellness Services Marketer & Manager Complete AI OS: Setup, Benefits & User Guide

**For:** wellness services marketers, marketing managers, program managers, community-growth leaders, small wellness-business operators and dual-hat service leaders<br>

## What THRIVE gives you

THRIVE turns a scattered AI chat habit into a governed working environment for truthful wellness marketing and humane service management. It helps the user define mission and career direction; verify claims and sources; design accessible human-voice content; understand approved aggregate audiences; build community and partner plans; coordinate campaigns, editorial review, aggregate demand/service/team capacity, SOP questions and handoffs; model value/budget/offer/vendor scenarios; learn from aggregate outcomes; and govern optional agents.

The embedded Mission Control provides four preview launchers, a seven-item attention ceiling, claim/consent/privacy/source controls, approval queues, service/team capacity and SOP questions, agent registry/kill controls, receipts, pause/reset/uninstall and an accessible Markdown fallback. It never becomes a CRM, ad platform, booking system, clinical system, HR system, finance system or publication engine.

## Benefits

- **Truth before reach:** exact claims remain tied to evidence, limitations, reviewer and expiry.
- **A more human brand:** plain-language and anti-AI-slop checks keep drafts specific, useful and accountable.
- **Better management:** aggregate workload/capacity, RACI, SOP and review queues become visible without employee surveillance.
- **Integrated planning:** campaign, content, service capacity, accessibility, finance and human gates connect without autonomous execution.
- **Ethical relationships:** community, partner, referral and testimonial concepts include reciprocity, consent, rights and neutrality.
- **Learning without vanity:** experiments start with the decision, baseline, guardrails, balancing measures and honest uncertainty.
- **A life beyond work:** private mission, career and protected-life planning stays in an owner-only isolated store.
- **Governed agents:** every optional agent is P0 Disabled and must pass ORBIT and independent fixtures before one bounded run.

## Before you install

1. Use a Hermes workspace you control and confirm Nurse AI OS is available.
2. Do not paste PHI, patient/client records, employee records, consumer-sensitive data, lead/contact lists, credentials or live campaign exports.

4. Save a copy of the installer. Installation may take a while and may require multiple visible Hermes turns; nothing runs in the background.

2. Open `Wellness-Services-Marketer-and-Manager-Complete-AI-OS-with-THRIVE-SuperPowers-Hermes-Program.md` and provide the entire file in one Hermes conversation.
3. Issue only `INSPECT THRIVE INSTALLER ONLY — CREATE NO STATE.` Do not use the installation prompt yet.
4. Review the exact combined THRIVE Activation Card, including the computed complete-program hash, target, authority, data/claim/action boundaries, components, states, burden, rollback, and unsupported controls.
5. Approve, revise, install the foundation only, or cancel. Silence, timeout, download, title, access, credentials, earlier work, or a general publish/install request is never approval.
6. Only after accepting the exact card, issue `INSTALL THRIVE AFTER S0 — USE EXACT VERIFIED COMPONENT HASHES AND KEEP ALL POWERS AND AGENTS INACTIVE.`
7. Keep the session open through S1 (40 criteria) and S2 (120 criteria). If paused, request the last receipt and resume; do not ask Hermes to claim background work.

## How launchers behave

A launcher opens a governed form and workflow preview. It accepts only the stated input class, produces a draft/brief, pauses at a human gate, creates a receipt and offers safe discard/rollback. It does not send, publish, spend, contact, book, refer, transact, connect, write to an official record or activate an agent.

## Pause, repair and uninstall

Pause stops proposed work and agents; Reset View changes presentation only. Repair verifies the canonical 17 component hashes, schema bundle/generator hashes and one dashboard. An approved export contains only explicitly selected nonsensitive records. Uninstall purges eligible THRIVE routes, schemas, state and optional HTML while preserving other lanes. An authorized external legal/records hold must be reported as retained state; THRIVE may not call that a complete uninstall or total purge.
