# Hermes HIPAA Posture

For this course, Hermes remains a no-PHI personal and professional learning environment unless an institution deliberately deploys it under formal governance.

Do not describe Hermes, Google Workspace, or any AI system as HIPAA-compliant just because it is paid, enterprise, encrypted, or self-hosted.

Compliance requires:

- correct environment
- access control
- auditability
- encryption
- risk analysis
- written policies
- workforce training
- vendor review
- BAA where required
- explicit organizational approval

## Course rule

```text
Hermes Desktop for this course = personal/professional learning, no PHI.
```

## Institutional rule

```text
PHI-capable AI requires privacy, security, legal, IT, clinical governance, BAA/vendor review, and documented approval.
```
