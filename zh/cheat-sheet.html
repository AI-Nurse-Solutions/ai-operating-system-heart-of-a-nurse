操作流程 — 安装与配置

Hermes 的菜单会随版本更新而变化。若界面与本页不同，请以 官方文档 为准。

安装 Hermes

访问官网 — hermes-agent.nousresearch.com — 下载适用于你电脑的版本，遵循其步骤。打开后，当设置询问 AI 模型或供应商时，先看下一步 —— 有免费及低成本方案。

权限 —— 给个房间，别给整栋楼

当 Hermes 请求文件访问时，仅授权你的 My-Nurse-AI-OS 文件夹。拒绝更宽泛的权限 —— 全盘、桌面、下载、照片。把它想成支援护士的工牌：只进入被分配的病区，而不是全院每个楼层。误授过宽？可在电脑的隐私设置中撤销。

选择 AI 供能方式 — 有免费及低成本路径

Hermes 本体免费且开源。 可能产生的费用来自驱动它的 AI 模型 —— 可以把 Hermes 想成免费的输液泵，模型像泵中输注的液体，按使用量计费。你有两条路：

路线 A — Nous 订阅。 最简：一次订阅，一切搞定。设置时直接勾选，随后跳过本节余下内容。

路线 B — 自带密钥。 最经济，也是以近乎零成本运行 Hermes 的方式。在 Hermes 设置中，选 完整设置（非默认订阅选项），再选 自带密钥。我们推荐 OpenRouter —— 免费账户、一枚 API 密钥、含免费及超低成本模型的完整菜单。具体操作：

  1. 在 openrouter.ai 注册账户，从个人面板生成 API 密钥。
  2. 先在 OpenRouter 设定消费上限 —— 这是防止意外账单的安全护栏。起点可设为几美元，之后随时可以上调。
  3. 在 Hermes 设置中，选 OpenRouter 作供应商，粘贴你的密钥。
  4. 选择模型时，先试免费或低成本选项。让 Hermes 处理一项简单任务，确认能够正常工作；如果回答质量不足，再切换到更强的模型。测试模型只按实际使用的 token 计费，不另收测试费。

警告： ChatGPT Plus 或 Grok 订阅通常不含 API 访问 —— 那部分需另算费用。最稳妥做法：一枚专属 OpenRouter 密钥，配上你自己设定的消费上限。

提交入门包

解压 入门包 至好记之处（Documents 为佳选），然后告诉 Hermes：

我的 Nurse AI OS 文件夹在 [位置]。这里存放我的文件 —— SOUL 文件、Memory 文件夹里的笔记、Projects 文件夹里的项目、Skills 文件夹里的可复用提示词。若那三个文件夹任一缺失，请创建。请把你为我做的工作放在此文件夹内。

构建仪表盘 —— 并添加书签

使用我的 SOUL 文件，个性化入门包中的 Local HTML Life Dashboard（08-Integrations/Local-HTML-Life-Dashboard）。填入我的名字、各维度首要优先级、测验所得首批目标与习惯。另存为 My-Dashboard.html 到我的 Nurse AI OS 文件夹，并告诉我确切位置。

在浏览器中打开生成文件并加为书签 —— 这就是你的站务板。

为什么 SOUL 文件和入门套件比其他内容都重要？ 强大的 AI 就像一匹精力充沛的骏马 —— 能力出众，但没有缰绳就可能失控。这两个文件就是缰绳。SOUL 文件告诉 AI 你的价值观、边界和绝不能越过的红线；入门套件则为它提供明确的空间与结构。两者结合，让系统拥有护士之心，也让控制权始终在你手中。跳过它们不会更快，只会让你更容易被系统牵着走。
通过两份设置指南继续深入： Hermes 基础指南（适合护理专业学生和重视预算的临床护士，帮助你以近乎零成本运行）和 Hermes 进阶指南（适合使用配置文件、定时任务和模型路由的进阶用户与领导者）。两份指南都说明 EDENA — Ethical Design & Enablement for Nursing Augmentation（护理增强的伦理设计与赋能）这一良知层如何在你的架构中实际运作。

操作后维护

使用系统 — 直至成为肌肉记忆

你不需要背下所有功能。先在脑中熟悉流程，操作自然会跟上 —— 就像你掌握的每项技能最终都会成为本能。养成以下七个习惯：

1 · 先为任务分诊，再选择模型

简单任务 —— 快速问答、摘要、改写、头脑风暴 —— 使用免费或低成本模型。复杂任务 —— 多步骤规划、长文档、编程、深入排障 —— 使用推理能力更强的模型。需要查看截图或图表时，选择支持视觉输入的模型。就像不会为处理一根木刺启动抢救团队：先从低成本选项开始，需要时再升级。

2 · 一个主题一个会话 — 就像一个患者一份病历

切勿将财务、副业、CCRN 复习塞入同一对话 —— 细节易混，答案易糊。专注对话产出更清晰。话题更迭？立刻新会话。

3 · 项目即护理计划；会话即工作班次

一次会话 = 一次对话。进行中的事项 —— 课程开发、商业计划、家庭项目 —— 用项目：工作区让文件、目标、对话长期共存，避免重复起步。

4 · 请教专家 — 用人设

吩咐 Hermes 扮演谁，其回应的深度与风格随之改变："以平实导师身份，逐步解释此错误。" · "以创业策划家身份，审阅我的计划以寻找风险及下一步。"

5 · 有疑问？问助手自己

你不需要完美指令。"接下来最佳行动是什么？" · "在继续前，问我你所需的问题。" · "把这事拆成易行步骤。" · "我该用哪个模型处理此任务？"

6 · 用截图排障 — 让画面说明问题

把截图当作排障数据，不要只靠文字描述。分享截图时同时说明：你原本想做什么、实际发生了什么、完整报错，以及已经尝试过哪些方法。截图中绝不能包含 API 密钥或密码 —— 必要时先裁剪或模糊处理。

7 · 管控开支

像管理耗材一样留意费用：开启消费上限，草稿使用低成本模型，把能力更强的模型留给真正需要复杂推理的任务；除非任务确实需要，否则不要上传大型文件。

日常流程 —— 练到成为本能： 按任务选择模型 · 新话题开启新会话 · 持续事项使用项目 · 需要专业视角时指定角色 · 有疑问就提问 · 卡住时提供截图 · 仅在必要时升级模型。
