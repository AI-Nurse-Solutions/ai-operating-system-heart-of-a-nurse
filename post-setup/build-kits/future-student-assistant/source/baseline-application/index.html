    01

    Start here

    DISCOVER Mission Control is a local, governed solution laboratory for the Nurse AI OS. It combines 24 SuperPowers, 24 bounded workflows, multiple role dashboards, a repeatable five-stage Mission Loop and evidence-based capability development. It is not an EHR, clinical decision tool, credentialing system or autonomous agent.

    1. Read README-FIRST, Privacy and Security, then open this package locally.
    2. Open this dashboard locally.
    3. Import an approved derived Soul Profile when available and select your role views.
    4. Start a non-sensitive mission or open the synthetic example.
    5. Use a mission stage or workflow card to create a Preview Only Hermes handoff.
    02

    Install on your computer

    Keep the entire downloaded folder together. Use the included Mac, Windows or Linux launcher; direct opening of index.html is a fallback. No cloud account or internet connection is required.

    Important: Browser storage is device-local but not encrypted. Session-only missions disappear when the page is closed. Locally saved missions must remain non-sensitive. Never treat this application as an approved clinical, research, employment or academic record.
    03

    Discover Packet, Soul Profile and privacy

    Mission Control accepts only a strict, derived Discover Packet configuration and a versioned, derived Soul Profile. Review each preview before applying it. Raw interview notes and raw quiz answers are not imported or stored. Packet settings may tune goals, workflow order and safe defaults; Soul results may tune presentation and role recommendations. Neither can verify competence, grant authority, enable an agent or weaken governance.

    04

    Coordinated role dashboards

    Open My Role Dashboards and select primary, supporting, emerging or contextual views. The top role switcher changes the active dashboard while the Shared Soul foundation remains one. Advanced Studies is a companion overlay. Roles are self-selected navigation labels—not verified credentials or delegated authority.

    05

    Start and manage a mission

    1. Name a non-sensitive goal, desired outcome and owning role.
    2. Choose session-only or local non-sensitive retention.
    3. Classify the context with the EDENA advisory and artifact state.
    4. Work through Assess, Define/Diagnose, Plan, Implement and Evaluate.
    5. Save, revise, pause or return to an earlier stage. Completing Evaluate enables a new iteration.

    Later stages may be explored, but their checkboxes are evidence of user review—not external approval. The dashboard cannot perform or verify real-world execution.

    06

    EDENA advisories

    Green is bounded and low consequence. Yellow requires verification. Orange requires structured qualified review. Red signals serious potential consequence or missing authority.

    In Personal Edition, Red permits sanitized exploration only after deliberate acknowledgment; it never authorizes real use. The Institutional policy preview demonstrates stricter stops but is not tamper-resistant institutional enforcement. A managed Institutional Edition would require centrally controlled policy, identity, audit and approval receipts.

    07

    Capabilities and badges

    Capability progress comes from sanitized evidence tied to evaluated missions, reviewed artifacts, measurements, feedback, reflections, safety drills and governed agent records—not clicks. Levels are Basic, Intermediate, Advanced and AI Agent Orchestration Master. They are development records, never licenses, credentials, CE credit or authority.

    08

    Use with Hermes and Nurse AI OS

    1. Open a workflow card and choose Prepare Hermes handoff.
    2. Use only public, synthetic or approved aggregate metadata. Keep the optional context box non-sensitive.
    3. Copy or download the generated Markdown prompt.
    4. Paste it into the DISCOVER lane in Hermes and wait for a visible Preview response.
    5. A human reviews the exact output before any decision or official-system action.

    The dashboard has no hidden Hermes connection. It never sends a prompt by itself. The included integration installer can ask Hermes to register a local launch link and Guide entry where the platform supports them.

    Safety boundary

    • No PHI, participant-level data, confidential research, credentials, secrets or protected personnel information.
    • No clinical, research, regulatory, financial, safety or employment determination.
    • No submission, publication, recruitment, consent, enrollment, deployment, purchase, send or official-system write.
    • No role selection or Soul result grants authority.
    • All AI-agent concepts remain disabled until separately authorized, bounded and tested by accountable humans.
    11

    Backup, update and uninstall

    Export the dashboard profile before replacing this folder and review the file because saved mission text is not encrypted. Import only into a trusted compatible release. Reset clears this app's local browser data. Uninstall by erasing local data, removing the dashboard registration from Hermes and deleting this folder. Use the included rollback instructions if an update fails.

    12

    Troubleshooting

    The page looks unstyled.
    Keep index.html beside the assets folder, then reopen it.
    Copy does not work.
    Some browsers restrict clipboard access for local files. Select the generated prompt manually or use Download Markdown.
    Personalization disappeared.
    Private browsing, browser cleanup or a different browser profile may remove localStorage. Restore a reviewed exported profile.
    Hermes cannot open the dashboard.
    Give Hermes the exact local path and use the included integration installer. If local links are unsupported, keep a desktop shortcut and use copy/paste handoffs.
