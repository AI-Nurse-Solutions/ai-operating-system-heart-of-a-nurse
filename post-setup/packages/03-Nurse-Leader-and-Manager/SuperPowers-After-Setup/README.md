# Hermes–Nurse AI OS SuperPowers Pack

## Start here

The SuperPowers Pack is a post-setup capability layer for an existing Hermes–Nurse AI OS. It gives the user reusable skills, guided workflows, dashboard launchers, and carefully bounded automations while preserving human authority, privacy, professional boundaries, and the right to stop.

> **Product promise:** SuperPowers do not make the AI more powerful over the user. They make the user more capable, clear, and in control.

This package contains:

- an always-on **Trust Shield**;
- the universal **Core Eight** SuperPowers;
- a canonical dashboard and launcher contract;
- a seven-day guided activation;
- five role-kit recipes;
- eight optional Power-Up packs;
- templates for missions, voice, decisions, workflows, and automation receipts;
- acceptance tests for privacy, quality, accessibility, automation, and human control.
- an illustrated Word user guide covering setup, daily use, troubleshooting, and safe customization.

## The four building blocks

| Building block | Meaning |
|---|---|
| **Skill** | A reusable capability, such as defining a mission, checking evidence, or improving clarity. |
| **Workflow** | A repeatable sequence that uses one or more skills, includes checkpoints, and produces a defined artifact or decision. |
| **Launcher** | A visible dashboard control that starts a named workflow and shows its mode, inputs, permissions, and approval requirements. |
| **Automation** | A proven workflow allowed to run within explicit boundaries, with monitoring, exception handling, activity history, and a stop or recovery path. |

The normal progression is:

`Prompt → Skill → Workflow → Launcher → Measured learning loop`

Automation follows a separate safety progression:

`Manual → AI-assisted → Preview → Approval-gated → Monitored automation`

## Installation

1. Confirm the user's base Hermes–Nurse AI OS is active and its identity, privacy, memory, no-PHI, and human-escalation rules have passed testing.
2. Give Hermes this entire package, beginning with `MASTER-INSTALLER.md`.
3. Instruct Hermes: **“Install SuperPowers Pack v1.0 in preview mode. Do not enable connectors, external actions, memory expansion, or automations until I approve the relevant activation cards.”**
4. Run the security preflight and persona overlay.
5. Complete the first-run conversation using synthetic or low-sensitivity information.
6. Pin no more than four launchers on day one.
7. Complete the seven-day quick start.
8. Review usefulness, burden, privacy, and trust before adding a role kit or Power-Up.

## Day-one launchers

Pin these four:

1. **Plan Today**
2. **Capture & Sort**
3. **Think It Through**
4. **Make This Human**

Keep **Privacy & Permissions** permanently visible as a system control. Introduce **Weekly Reset** after the first completed week.

## User quick start

Try these commands after installation:

- “Interview me and create a one-sentence mission, five principles, a not-do list, and three possible 90-day outcomes.”
- “Plan today around my current capacity, fixed commitments, protected personal time, and one Plan B.”
- “Capture this note. Separate facts, interpretations, decisions, and next actions. Show what you propose to remember.”
- “Create a decision brief. Show options, tradeoffs, reversibility, assumptions, and the human decision owner.”
- “Rewrite this in my voice and run the Human Standard. Do not invent details.”
- “Turn this into a one-screen, accessible dashboard with clear next actions.”
- “Observe this repeated task and propose a workflow. Keep it in preview mode.”
- “Run my weekly reset and recommend what to continue, change, pause, release, or verify.”

## Safety baseline

- Patient information and confidential healthcare information do not enter a general SuperPowers workspace.
- A relationship, job title, classroom role, or organizational position does not automatically grant data access or authority.
- The system drafts and stages consequential actions; the responsible human reviews and approves them.
- No unattended sending, publishing, spending, transferring, signing, filing, deleting, changing permissions, or making clinical, employment, legal, financial, academic-integrity, or other high-consequence decisions.
- Memory is visible, editable, purpose-limited, exportable, and deletable.
- A user can pause a workflow, hide a launcher, revoke a connection, or uninstall a Power-Up without penalty.

## Stewardship

- Review the pack quarterly and after any model, connector, permission, role, policy, or data-source change.
- Re-run the acceptance tests after changes.
- Retire workflows that create more burden than benefit.
- Keep version, owner, change log, maximum automation level, and review date visible for every installed module.

**Final principle:** Greater clarity, craft, recall, and follow-through must come before greater autonomy.
