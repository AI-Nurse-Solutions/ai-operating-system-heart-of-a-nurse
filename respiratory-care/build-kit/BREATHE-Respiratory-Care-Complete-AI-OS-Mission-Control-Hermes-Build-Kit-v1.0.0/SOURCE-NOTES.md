
# Source notes

The canonical Respiratory Care Professional documents use `Implementation-ready` to describe written programs. That is not runtime evidence. The target remains not operational until Hermes builds the application and executes its tests.

The standalone BREATHE directory and its archived ZIP differ in their README and manifest, and the standalone directory manifest references setup guides not present there. Both variants are retained unchanged as provenance. The later integrated Complete Edition program and setup guide control the one-package installation sequence.

The route, product ID, target app version, foundation namespace, machine schemas, data-class codes, stable BREATHE IDs, agent registry, permission vocabulary, capability system and control ledger are implementation-generated additions. They grant no authority.
