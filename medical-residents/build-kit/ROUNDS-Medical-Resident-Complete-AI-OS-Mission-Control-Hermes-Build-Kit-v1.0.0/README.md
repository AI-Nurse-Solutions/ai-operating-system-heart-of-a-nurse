
# ROUNDS — Medical Resident Complete AI OS Mission Control — Hermes Functional Build Kit

`NAIO-MR-ROUNDS-FUNCTIONAL-BUILD-KIT-1.0.0` is a source-pinned, self-verifying implementation handoff for building a downloadable local `My ROUNDS` application.

## Included

- immutable functional-build prompt, DISCOVER v2 baseline, ROUNDS source pack, complete one-file program, setup guides, and historical package;
- exact Core Four, 24-power, 24-workflow, 30-template, 10-agent, and 17-record inventories;
- strict Discover, Soul, and Mission Profile adapters with synthetic fixtures and complete field provenance;
- ATTEND, CIRCLE, ORBIT, EDENA, data-admission, supervision, human-authority, and no-execution contracts;
- four-level capability system with 17 domains and 77 criteria;
- 216 implementation controls, 48 full-stack scenarios, and all 160 canonical ROUNDS checks—424 runtime records;
- user Guide, sample mission, security checklist, installation, backup, restore, update, rollback, ROUNDS removal, and uninstall guidance;
- release manifest, checksums, and an embedded verifier.

## Boundary

This is not an EHR, sign-out, patient list, clinical decision-support, diagnosis, treatment, prescribing, ordering, documentation, paging, consultation, handoff, coding, billing, claims, credentialing, resident evaluation, QI/research determination, or autonomous action system. Possible patient or restricted content is rejected before echo, transformation, persistence, logs, memory, export, model, agent, tool, or backup use.

Run `python3 tools/verify-build-kit.py --package .`. Passing verifies this handoff only; Hermes must still build and execute the target.
