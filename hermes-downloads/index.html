Plain language

What open-source Hermes is to Nurse AI OS

Hermes Agent is an open-source AI agent framework by Nous Research. It can run in a terminal, desktop workflow, messaging platforms, and tool-enabled environments. It can use memory, skills, tools, web access, files, and automations depending on how the user configures it.

Engine

Hermes is the runtime

Think of Hermes as the engine that can read, write, search, remember, call tools, and carry out tasks when a human permits it.

Stewardship

Nurse AI OS is the governed layer

Nurse AI OS gives Hermes a nursing-centered purpose: no-PHI boundaries, SOUL files, EDENA gates, dashboards, prompts, starter folders, and post-setup pathways.

Human gate

The nurse remains the judge

Hermes may propose, organize, summarize, or draft. The nurse decides what is true, safe, appropriate, and ready to use.

The simplest metaphor: Hermes is the capable open-source assistant. Nurse AI OS is the nurse-led operating discipline that tells the assistant what it may do, what it must never do, and how to serve human dignity.

What we provide

After Hermes is installed, Nurse AI OS gives it a safe bedside posture.

Nurse AI OS is not a replacement for Hermes and not a separate clinical system. It is the nurse-centered harness around your AI work: values, boundaries, folders, dashboards, prompts, weekly review, and governance language.

Recommended sequence

Set it up in this order.

  1. Open the official Hermes Agent site. Install Hermes using the current official instructions from Nous Research. Go to Hermes →
  2. Return to Nurse AI OS. Do not try to customize everything at once. Start with the nurse-facing setup path, not random online prompts.
  3. Take the SOUL Quiz. It drafts how you want AI to work with you, where your boundaries are, and what judgment must remain human. Take the SOUL Quiz →
  4. Download the Nurse AI OS starter kit. Use it for folder structure, no-PHI boundaries, starter prompts, dashboard files, and first-week steps. Download starter kit →
  5. Open the Local HTML dashboard first. Save it locally, bookmark it under Dashboards, and use broad strokes now. Open HTML dashboard →
  6. Use Notion later if you need a relational command center. Notion is powerful, but it is an advanced option after the local dashboard makes sense.
  7. Only then explore the governed OS path. Advanced harnesses, dry runs, recovery, and institutional workflows come after the no-PHI foundation is stable. Read the governed OS guide →
Dashboard choice

Start with Local HTML. Move to Notion only when you need more structure.

The dashboard should lower cognitive load, not become another system to maintain. For launch, the Local HTML dashboard is the first recommendation because it is the least fragile path for tired nurses.

Recommended first

Local HTML Life Dashboard

A single-file, browser-only dashboard for health, finances, goals, habits, tasks, and routines.

Advantages

  • No account required.
  • No server, cloud workspace, or database setup.
  • Works as a simple local file in the browser.
  • Fastest path for users who are new to AI tools.
  • Easy to bookmark under Dashboards.
  • Exports/imports a JSON backup when the user is ready.

Open Local HTML dashboard →

Advanced option

Notion Life Dashboard Pack

A database-style no-PHI dashboard pack for users who want relational views, linked tables, and more structured operations.

Advantages

  • Better for users already comfortable with Notion.
  • Supports linked databases, filters, views, and relations.
  • Useful for educators, leaders, or builders managing multiple projects.
  • Easier to expand into a shared command center with explicit governance.
  • Stronger when the user needs dashboards across goals, content, pilots, and reviews.

Open Notion setup guide →

Rule of thumb: Local HTML is for first safety and momentum. Notion is for advanced structure after the user knows what they actually need to track.

The safety perimeter

No PHI. No clinical decisions. No silent automation.

Do not enter patient names, charts, screenshots, identifiers, room numbers, employer secrets, passwords, account numbers, or patient-specific clinical details into Hermes, the quizzes, dashboards, or any AI tool unless an approved institutional environment and accountable governance explicitly allow it.

Hermes can help with learning, organization, drafting, research support, project planning, and reflection. It does not carry a license, validate competency, approve clinical deployment, replace educators, or make patient-care decisions.

Agents propose. Humans judge. Nurses steward.
