# Research Skill Prompts

Use these after a workflow has worked manually 2–3 times. Skills should standardize a proven method, not automate an untested habit.

## capture-article

```text
Create a reusable skill called capture-article.
Purpose: turn one article, essay, newsletter, or web page into an Obsidian-ready learning note.
Workflow: understand the source, extract key ideas, note claims and uncertainty, connect to related topics, and create follow-up questions.
Boundaries: no PHI, no confidential employer material, no patient-specific recommendations.
```

## read-paper

```text
Create a reusable skill called read-paper.
Purpose: summarize academic or medical papers as evidence notes.
Output: question, population, methods, findings, limitations, uncertainty, confidence, practical implications, and source link.
Boundary: summarize evidence only; do not give clinical advice or patient-specific recommendations.
```

## repo-brief

```text
Create a reusable skill called repo-brief.
Purpose: inspect a project repo or GitHub context and write a technical research brief.
Output: repo purpose, key files, open questions, risks, options, evidence links, and recommended next step.
Boundary: do not push, publish, alter code, or create issues without approval.
```

## weekly-digest

```text
Create a reusable skill called weekly-digest.
Purpose: review new notes from the past seven days and create a short digest.
Output: what changed, top ideas, connections, unresolved questions, and next week's learning focus.
Boundary: non-PHI only; do not access employer systems or clinical records.
```
